How I completed this assignment

The repo for this assignment can be found here.

The azure deployment for this assignment can be found here.


I started off by evaluating the sample structure given to us for the assignment. The structure we were supposed to follow featured the following:

  • 9 unique elements assigned to the page
  • An image, which was to be hard coded in
  • An Edit button, which would allow someone to adjust each of the 9 unique elements.

After identifying these core components, I created a new MVC project in Visual Studio. I decided to create a similar layout, but for software products.

After creating a new MVC project, I started by creating the model for our objects. I named it Game.cs, and gave it the following attributes:

  • Title
  • ReleaseDate
  • Genre
  • Price
  • esrbRating
  • Description
  • Developer
  • Publisher
  • Score

After completing the model, I generated a new controller based on the model. Creating this controller also generated the views I needed to work with the data (Index, Details, Create, Edit, Delete.)

Once this was finished, I edited the HTML inside of the Details view to match the layout of the assignment specifications as closely as possible. From the Details view, there is an Edit button that will allow you to edit the object.

I also made sure to seed the database with test data to make sure that it doesn't throw errors when running for the first time.

The project has been deployed to azure, but for some reason, azure is giving errors when attempting to access any views past the home page. For more details about what I encountered, see below.




Locally, the project runs perfectly. Once deployed to azure, attempting to view the Index/Details/Edit/Delete views all throw a generic error. Going to the Create view works, but attempting to create an object also throws the same generic error.

This seems to happen regardless of whether I connect the site to a database in azure while deploying it. It's throwing generic errors for security reasons, so I went into my web.config file and added "customErrors mode="Off"" to the appropriate area so that it would display errors that were more specific.

However, adding this one line of code to the web.config file caused the website to not work at all once deployed to azure. Instead, navigating to the homepage would only result in azure's generic "This Web App has been Successfully Created" web page, instead of the homepage of my project. Undoing that line of code in the web.config file does fix this problem, but also leaves me back where I started.

I've not yet found a solution to this issue, so I attached an image of my layout below so that you can compare it to the assignment requirements. Editing the object does work, and the changes do persist. The code is also available in the github repo.
